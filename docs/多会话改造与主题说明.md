# 多会话改造与主题说明

## 本次主要工作

本次改造的目标，是把原来的单轮聊天页面升级成一个支持多会话管理的 AI 应用，同时尽量保留项目原本的交互节奏和视觉风格。

本次完成的内容包括：

1. 将单一聊天状态升级为多会话状态管理。
2. 将聊天路由升级为按会话 ID 访问的结构。
3. 新增会话侧栏，支持新建、切换、删除会话。
4. 每个会话拥有独立的消息列表、上下文历史、输入草稿、图片草稿和流式状态。
5. 增加本地持久化，刷新页面后可以恢复会话。
6. 将主题切换升级为可选主题面板，支持 `深空`、`明亮`、`日暮`、`森林` 四套主题。
7. 调整输入区布局与视觉比例，使输入框回到页面底部居中位置，会话侧栏密度更轻。

## 改造后的核心结构

### 1. 路由层

现在的聊天路由使用：

```text
/chat/:conversationId
```

这样做的好处是：

1. 当前会话有稳定地址。
2. 刷新页面时可以直接恢复指定会话。
3. 会话切换不再依赖单纯的前端临时状态。

相关文件：

- `src/route/router.tsx`
- `src/App.tsx`

### 2. 状态层

原本项目中只有一份全局聊天状态，例如：

- `messages`
- `chatHistory`
- `isStreaming`

现在改为：

```ts
conversations: Record<string, Conversation>
currentConversationId: string | null
orderedConversationIds: string[]
abortControllers: Record<string, AbortController | null>
```

每个 `Conversation` 包含：

- `id`
- `title`
- `messages`
- `chatHistory`
- `draftInput`
- `uploadingImages`
- `isStreaming`
- `createdAt`
- `updatedAt`
- `lastMessagePreview`

相关文件：

- `src/store/chatStore.ts`
- `src/types/chat.ts`

### 3. 发送链路

发送消息时，逻辑不再操作“全局聊天”，而是始终操作“当前会话”。

发送流程如下：

1. 读取当前 `conversationId`。
2. 校验文本或图片是否为空。
3. 读取 API Key。
4. 先写入当前会话的用户消息。
5. 再写入当前会话的模型历史。
6. 创建 assistant 占位消息。
7. 发起流式请求。
8. 持续更新该会话中 assistant 消息内容。
9. 请求完成后，把回复写回该会话历史。

相关文件：

- `src/hooks/useSendMessage.ts`
- `src/services/api.ts`
- `src/utils/messageContent.ts`

## 会话管理是如何实现的

### 会话创建

调用 `createConversation()` 时：

1. 生成唯一 ID。
2. 生成默认标题 `新会话`。
3. 初始化空消息列表和空上下文。
4. 插入到 `orderedConversationIds` 头部。
5. 设置为当前会话。

### 会话标题

当前策略是：

1. 优先取首条用户问题。
2. 截断为较短标题。
3. 作为侧栏显示内容。

这样侧栏就可以只展示“问题本身”，不再重复展示时间和摘要。

### 会话切换

切换会话时，只更新 `currentConversationId` 和路由。
页面上的消息列表、输入草稿、图片预览、流式状态都从对应会话对象中读取，因此会自动切换。

### 会话删除

删除会话时会做几件事：

1. 清理该会话的图片预览 URL。
2. 中止该会话可能存在的流式请求。
3. 从会话字典和排序列表中删除。
4. 如果删除的是当前会话，则自动切到剩余会话或创建新会话。

## 主题系统是如何实现的

### 1. 主题模型

主题现在不再是单纯的明暗二元切换，而是使用：

```ts
type ThemeMode = "dark" | "light" | "sunset" | "forest";
```

### 2. 主题变量

所有主题都通过 CSS 变量驱动，定义在：

- `src/styles/theme.css`

每套主题都覆盖以下变量：

- `--bg`
- `--bg-grad-1`
- `--bg-grad-2`
- `--text`
- `--text-soft`
- `--line`
- `--brand-start`
- `--brand-end`
- `--input-bg`
- `--bubble-user`
- `--bubble-ai`

### 3. 主题切换

输入区右侧新增主题面板按钮，点击后可直接选择主题。
选中的主题会：

1. 写入 Zustand store。
2. 同步写入 `localStorage`。
3. 通过 `document.body.dataset.theme` 驱动页面实时换肤。

相关文件：

- `src/components/Composer.tsx`
- `src/hooks/useAppEffects.ts`
- `src/store/chatStore.ts`

## 输入区和布局调整

这次根据交互反馈做了两类回调式优化：

### 1. 输入框位置回调

输入框恢复到底部居中固定布局，不再为了适配会话侧栏而整体右移。

实现位置：

- `src/styles/composer.css`

### 2. 比例协调

聊天页改成较轻的双栏比例：

- 左侧会话栏：`280px`
- 右侧主聊天区：自适应剩余宽度

实现位置：

- `src/styles/base.css`
- `src/styles/chat.css`
- `src/styles/responsive.css`

## 持久化设计

使用 Zustand 的 `persist` 中间件保存：

- 当前主题
- 当前会话 ID
- 会话顺序
- 会话消息和历史
- 输入草稿

不会持久化的内容：

- `AbortController`
- `uploadingImages`
- `isStreaming`

原因是这些内容属于运行态对象，不适合直接序列化保存。
